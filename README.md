# BookWorms

Веб-застосунок на **Django** для читачів: стрічка постів, персональні полиці книг, пошук по полицях інших користувачів, **позика** та **обмін** книгами, приватні повідомлення в контексті запитів.

---

## Що реалізовано (бізнес-логіка)

### Облікові записи та профіль

- Реєстрація з **підтвердженням електронної пошти**: обліковий запис створюється неактивним (`is_active=False`), після переходу за посиланням з листа — активація.
- Вхід / вихід (сесійна автентифікація Django).
- Профіль (`/profile/`): перегляд; редагування — **біографія** та **аватар** (завантаження файлу). Колекція дефолтних аватарів — через адмінку (`AvatarCollection`).

### Книги та "Моя полиця"

- Книга в системі - **довідник за унікальним ISBN**: одна сутність `Book` на весь сайт (назва, автори, видавець, дати, обкладинка за URL, посилання на Open Library).
- Додавання на свою полицю (`Shelf`):
  - за **ISBN** з підвантаженням метаданих з **Open Library**;
  - або **вручну** (усі поля, включно з ISBN), якщо книги немає в API.
- На одній полиці одна й та сама книга не дублюється (обмеження `user` + `book`).
- Для книги задається **рекомендований вік читача** (діапазон 0–18, 18 трактується як "18+"); на сторінці полиці є **блокування повторного редагування** віку до "розблокування" (сесія).

### Полиці інших і пошук

- Перегляд полиць інших користувачів, окрема **публічна полиця** користувача.
- **Пошук** записів `Shelf` у чужих полицях за полями книги: назва (`q`), ISBN, видавець, дата видання, автор (з довідника), діапазон рекомендованого віку (перетин інтервалів). Для залогінених — панель у навігації зі швидким вибором книги / авторів / видавців.
- **Історія книги** — хто має цей запис `Book` на полиці (агрегований перегляд).

### Обмін і позика

- **Запит** щодо конкретного рядка чужої полиці (`BookExchangeRequest`):
  - **без книги взамін** — після прийняття це **позика**: у позичальника з’являється книга з полем "позичено у" (`borrowed_from`), власник не втрачає запис до повернення;
  - **з книгою зі своєї полиці** — **обмін**: дві книги переходять між власниками (без режиму позики).
- Не можна запитувати **свою** книгу, книгу, яку вже **тримають як позичену**, позичену **собі** для обміну тощо (перевірки в `exchange_service`).
- Статуси запиту: очікує, прийнято, відхилено, скасовано. **Пакетне** створення кількох запитів за одну дію.
- **Повернення позики**: позичальник ініціює повернення - у власника **підтвердження** (`return_pending` / `confirm_return`), після чого книга знову на полиці власника.

### Повідомлення

- **Приватні повідомлення** між користувачами; можуть бути прив’язані до запиту на обмін для контексту. Окремий **тред** з партнером (`/messages/<id>/`), позначення прочитаних.

### Пости (стрічка)

- Пости з заголовком і текстом; опційно — **посилання на книгу** з довідника (наприклад, про прочитане з полиці).
- **Коментарі** та **лайки** (один лайк від користувача на пост).
- Автор може **створювати, редагувати, видаляти** лише свої пости.
- На головній — **пагінація** (5 постів на сторінку), фільтр "лише мої пости" для залогіненого.

### Адміністрування

- Стандартна Django **admin** для моделей (книги, полиці, запити, користувачі тощо).

## Технології

| Шар | Стек |
|-----|------|
| Backend | Python, **Django 6**, ORM, шаблони, `django.contrib.auth` з кастомним користувачем |
| Frontend | HTML, CSS, **Bootstrap 5**, мінімальний JS у статику |
| Зображення | **Pillow** (аватари, `ImageField`) |
| БД | **SQLite** локально (`db.sqlite3`); у продакшені — **Azure SQL** через `mssql-django` + ODBC, якщо задано `AZURE_SQL_*` у середовищі |
| Статика / прод | **WhiteNoise**, **Gunicorn** (для Linux/Azure) |
| Конфіг | **python-dotenv** (`.env`), змінні `DEBUG`, `ALLOWED_HOSTS`, `CSRF_TRUSTED_ORIGINS`, `DJANGO_SECRET_KEY` |

Листи реєстрації йдуть через **SMTP** (у репозиторії можуть бути приклади налаштувань - для реальної пошти замініть на свої облікові дані / Mailtrap).

## Запуск локально

1. Клонувати репозиторій і перейти в корінь проєкту.
2. Віртуальне середовище та залежності:

   ```bash
   python -m venv venv
   venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. З каталогу `bookworms` (там лежить `manage.py`):

   ```bash
   cd bookworms
   python manage.py migrate
   python manage.py createsuperuser
   python manage.py runserver
   ```

4. Відкрити `http://127.0.0.1:8000/`.

За потреби створіть `bookworms/.env` з `DJANGO_SECRET_KEY`, параметрами БД для Azure тощо.

## Структура репозиторію (коротко)

- `bookworms/` — проєкт Django (`settings`, `urls`, `manage.py`).
- `bookworms/mainApp/` — основна логіка: моделі книг/полиць/обміну/постів, `openlibrary`, `exchange_service`, `book_search`, шаблони.
- `bookworms/profileApp/` — маршрути профілю під префіксом `/profile/`.
